# External Integrations

**Analysis Date:** 2026-06-16

## APIs & External Services

**Axon TMS (Trucking Management System):**
- Axon TMS web portal - Source of all dispatch/load CSV reports; scraped via Playwright browser automation
  - SDK/Client: `playwright` (Chromium headless)
  - Auth: username + password stored in user JSON config (`auth.username`, `auth.password`, `auth.base_url`)
  - Default URL: `https://catom.axoneta.io/`
  - Implementation: `execution/download_reports.py` — launches isolated Chromium context, logs in, navigates to reports, downloads CSVs

**Anthropic Claude API:**
- Claude `claude-haiku-4-5-20251001` - AI-powered diagnostic/error explanation feature in the app's help panel
  - SDK/Client: raw `urllib.request` with manual JSON construction (no SDK dependency)
  - Auth: `anthropic_api_key` in user JSON config or `ANTHROPIC_API_KEY` env var
  - Endpoint: `https://api.anthropic.com/v1/messages`
  - Header: `anthropic-version: 2023-06-01`
  - Implementation: `app/main.py` lines ~1318–1370

## Data Storage

**Databases:**
- None — no database. State is file-based: JSON config + CSV/XLSX files on local disk.

**File Storage:**
- Cloudflare R2 (S3-compatible object storage)
  - Bucket: `catom-updates` (env var `R2_BUCKET`)
  - Endpoint: env var `R2_ENDPOINT_URL` (Cloudflare account-specific)
  - Auth: `R2_ACCESS_KEY_ID` / `R2_SECRET_ACCESS_KEY` — baked into `app/feedback_credentials.py` at CI build time via GitHub Actions secrets
  - Two uses:
    1. **Update distribution** — installer `.exe` and `latest.json` update feed published to `s3://catom-updates/catom/` on tagged GitHub releases
    2. **Feedback bundles** — diagnostic zip uploads to `s3://catom-updates/feedback/<timestamp>-<id>.zip` triggered by in-app feedback button
  - Client: custom SigV4 implementation in `app/feedback.py` (no boto3; pure stdlib `hmac`/`hashlib`/`urllib`)
  - Public read URL base: `https://updates.aisimple.co/catom/`

**Caching:**
- None

## Authentication & Identity

**Auth Provider:**
- No user auth system. Single-tenant desktop app.
- Axon TMS credentials (username/password) stored in local JSON config managed by setup wizard.
- Anthropic API key stored in same local JSON config.

## Monitoring & Observability

**Error Tracking:**
- None (no Sentry, Datadog, etc.)

**Logs:**
- In-app log panel — `PipelineAPI._log()` streams lines to the UI via `window.pywebview.api` callbacks
- `last_run/` directory — last pipeline inputs/outputs written to disk; bundled into feedback zip for remote diagnosis
- Feedback bundle (`app/feedback.py`) — on user request, zips `last_run/` + app log + Distribution PDF + redacted config and uploads to R2

## CI/CD & Deployment

**Hosting:**
- Cloudflare R2 for binary distribution (fronted by `updates.aisimple.co` custom domain)
- GitHub Releases for `.exe` and `.dmg` artifacts

**CI Pipeline:**
- GitHub Actions — `.github/workflows/build.yml`
  - Triggers: `workflow_dispatch` or tagged `release`
  - Windows job: installs deps → bakes R2 credentials → PyInstaller build → NSIS installer → attaches to GitHub Release → publishes to R2
  - macOS job: installs deps → PyInstaller build → uploads artifact (no auto-update path; dev use only)

## Environment Configuration

**Required env vars / secrets (GitHub Actions):**
- `R2_ACCESS_KEY_ID` — Cloudflare R2 access key
- `R2_SECRET_ACCESS_KEY` — Cloudflare R2 secret key
- `R2_ENDPOINT_URL` — Cloudflare account R2 endpoint URL
- `R2_BUCKET` — R2 bucket name (default `catom-updates`)
- `GITHUB_TOKEN` — auto-provided; needs `contents: write` permission (set in workflow)

**Runtime config (user machine, JSON file):**
- `auth.base_url` — Axon TMS URL
- `auth.username` — Axon username
- `auth.password` — Axon password
- `anthropic_api_key` — Claude API key for error explanations

**Secrets location:**
- CI secrets: GitHub repository secrets
- Runtime secrets: local user JSON config (path varies by OS, managed by `app/main.py`)
- Build-time R2 credentials: `app/feedback_credentials.py` (generated by CI; not in source repo; excluded via `.gitignore`)

## Webhooks & Callbacks

**Incoming:**
- None

**Outgoing:**
- R2 PUT upload on feedback submission (fire-and-forget, no callback)
- Anthropic API POST on error explanation request (synchronous response)
- `https://updates.aisimple.co/catom/latest.json` polled on app startup for auto-update check (`app/updater.py`)
